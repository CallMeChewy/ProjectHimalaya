## Adding New Documents

When adding new documents to the Knowledge Database:

1. Create the document using the appropriate template
2. Add an entry to this index in the correct category
3. Include relative links to make navigation easier
4. Update the "Last Updated" field with the creation or modification date

## Code File Timestamp Requirements

All code files in the Project Himalaya ecosystem must maintain accurate creation and modification timestamps:

- **Format**: `# Created: YYYY-MM-DD` and `# Last Modified: YYYY-MM-DD  HH:MM{AM|PM}`
- **Spacing**: Two spaces must separate the date and time in the Last Modified timestamp
- **Time Format**: 12-hour time with AM/PM designation is required
- **Update Frequency**: The Last Modified timestamp must be updated with every code change
- **Verification**: Code review processes should verify timestamp accuracy

## Document Naming Conventions

- Use kebab-case for all document filenames
- Include descriptive names that clearly indicate content
- For versioned documents, include the version in the filename (e.g., `Document-Name-v1.2.md`)
- For date-specific documents, include the date in ISO format (e.g., `Document-Name-2025-03-15.md`)
